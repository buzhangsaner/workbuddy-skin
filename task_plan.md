## 关键决策

- 使用 WorkBuddy 内置 `WORKBUDDY_REMOTE_DEBUGGING_PORT`，不修改安装包。
- CDP 强制绑定 `127.0.0.1`，目标需同时通过 URL、标题和 DOM 指纹验证。
- 主题包只允许 JSON、CSS 和图片，不执行主题自带 JavaScript。
- Chat 首页使用 Hero；Terminal、Editor、Canvas 等工作区只应用轻量主题。
- 默认端口 `9336`，状态存储在 `%LOCALAPPDATA%\WorkBuddyDreamSkin`。
- 编辑器保持单文件、离线可用，只输出经过白名单约束的声明式主题 JSON。
- 图片只用于 Hero 背景，不允许包含伪造的菜单、按钮或文案。
- 主题目录由 `themes/catalog.json` 统一注册，Windows 与 macOS 共用相同 ID。
- 右上角切换器属于现有安全注入层，不修改 WorkBuddy 安装包，也不注册第三方 Codex 插件。
- “恢复原主题”只停用视觉覆盖并保留切换栏，桌面 Restore 仍负责完整卸载注入。
- 沉浸式改造只作用于 Chat 欢迎页；Terminal、Editor、设置等工作区保持轻量换色，避免影响生产力界面。
- 内置编辑器只接受文本、十六进制颜色和 PNG/JPG/WebP；不接受自定义 CSS、SVG 或 JavaScript。
- 自定义主题草稿只用于实时预览；取消恢复打开前选择，保存后才写入独立 localStorage 并将选择设为 `custom`。
- 场景选择器只通过主题 CSS 隐藏，不删除 WorkBuddy 原生节点；GitHub 图标使用内联 SVG 和固定 HTTPS 地址，不加载第三方图标脚本。

## 错误记录

| 错误 | 处理 |
|---|---|
| Payload 测试只预期转义 `<`，实现同时安全转义 `>` | 修正测试为完整 `\\u003c/script\\u003e` |
| 资源复制命令多写了一个 `esac` | 拆成顺序命令并重新执行安全路径检查 |
| agent-browser 的 Chromium 安装受本机 npm `ERR_INVALID_URL` 阻断 | 改用已安装 Chrome Headless 完成同等截图检查 |
| Windows PowerShell 5 将无 BOM 中文脚本误解析为语法错误 | 脚本与快捷方式名改用 ASCII，编辑器继续保留中文界面 |
| 恢复脚本先移除样式时，后台守护进程会竞态重新注入 | 调整为先精确停止注入器，再清理页面样式；实机复测通过 |
| 临时 Chrome 测试进程占用测试目录 | 按专用 user-data-dir 精确停止测试进程后完成清理 |
| 薛凯琪主题只是贴完整 UI 截图，且新版菜单/按钮未换色 | 生成无 UI 的纯人物背景；按实机稳定 class 重写导航、按钮和输入区状态 |
| PowerShell 5 将 JSON 主题数组作为单个嵌套对象 | 显式逐项展开目录，选择器恢复为 8 个独立选项 |
| 连续截图请求超时并堵塞 Chromium 截图队列 | 为 CDP 调用增加独立超时；截图放宽至 30 秒并重启后逐套验证 |
| 黑金只改到内层，外层网格与列表仍为白色 | 用坐标命中和计算样式定位 `.teams-container`、网格项与 `.conversation-list`，补齐深色覆盖 |
| 新增切换器测试首次运行 4 项失败 | 失败原因分别为 catalog payload、切换 DOM、胶囊 CSS 与守护尊重选择尚未实现，符合 TDD 预期 |
| 全量回归中旧热更新测试仍匹配 `imageDataUrl` | 对比实现确认数据源已升级为 `theme.imageUrl`，更新断言后保留相同热刷新行为验证 |
| 实机持久化检查动态拼接表达式触发 Renderer `Uncaught` | 前两次实际点击已成功；将 6.5 秒检查改为明确对象表达式并启用 shell 遇错停止，避免字符串切片构造语法 |
| 高速遍历主题时清透选择被旧红白刷新短暂覆盖 | 状态中 `scheduled` 是创建时的 null 快照，热重注入无法取消旧延迟任务；新增动态 getter 回归测试并修复 |
| 用户真实点击仍无响应 | 系统鼠标测试与 DOM 命中结果不一致；交互栏嵌套在 `pointer-events:none` 装饰根层，改为独立 `no-drag` 交互根节点 |
| 新增沉浸式首页测试首次运行 4 项失败 | 缺少品牌叙事 DOM、全屏 Hero、原生控件重排与响应式回退，符合 TDD 预期 |
| 热更新后品牌标题未出现 | 旧 `workbuddy-dream-chrome` 被复用但没有新节点；新增热升级补建逻辑和回归测试 |
| 黑金快捷卡被浅色玻璃规则覆盖 | 实机计算样式确认是选择器优先级不足；增加 Chat+黑金联合选择器 |
| 欢迎页输入框偏右、左栏漏色、对话页变回原生 | 实机 DOM 确认固定定位在变换容器内重复偏移 264px、`.conversation-list` 自带灰底、`.main-content--chat` 自带白底；分别修正定位并补齐侧栏与对话区覆盖 |
| 清透主题的场景按钮压住状态条 | 实机矩形确认状态条底部 y=383、场景栏顶部 y=378；改为相对功能卡底部定位并合并成分段玻璃栏 |
| 新增内置编辑器测试首次运行 5 项失败 | 缺少自定义主题模型、面板 DOM、图片压缩、响应式 CSS 和清理逻辑，符合 TDD 预期 |

| 保存验证脚本读取了旧的选择键名 | 对照实现改为 `workbuddy-dream-skin-selection-v1`；确认内容与选择均已持久化并完成完整重启验证 |
| `git ls-remote` 被本机错误代理 `127.0.0.1:58001”` 拦截 | 不修改用户全局配置，后续仅对本次 Git 命令显式禁用该代理后重试 |
| 首次测试补丁因主题栏测试末尾正则上下文不完全匹配而未应用 | 不重复原补丁；先读取文件尾部精确上下文，再拆分为两个小补丁 |

| GitHub CLI 最初只有失效的 `buzhangsan` 凭据且 SSH 无密钥 | 通过 GitHub 官方设备授权登录 `buzhangsaner`，清除单次进程代理后成功推送 `main`，未改全局代理 |
