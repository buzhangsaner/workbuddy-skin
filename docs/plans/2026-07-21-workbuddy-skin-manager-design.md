# WorkBuddy 皮肤管理器设计

## 目标

在现有 `workbuddy-dream-skin` 仓库中新增跨平台 Electron 管理器，为 Windows 与 macOS 用户提供主题卡片浏览、一键应用、完全还原、安全导入导出和自定义主题设计。管理器直接复用现有 WorkBuddy CDP 注入器、主题目录与选择持久化，不修改或重新分发 WorkBuddy 的 `app.asar`、客户端源码和专有资源。

内置主题由 9 套扩展到 15 套。新增“赛博霓虹、仙风翠玉、极光废土、机甲警戒、墨韵山河、海盐玻璃”6 套原创主题，每套使用无 UI、无文字、无 Logo、无受版权保护角色的纯背景素材，并覆盖欢迎页、侧栏、卡片、项目页、对话页和输入区的视觉色板。

## 架构

管理器位于 `manager/`，由 Electron 主进程、隔离 preload API 和静态 Renderer 三部分组成。主进程仅暴露固定 IPC：读取状态、应用主题、恢复原版、导入、导出、选择背景、生成主题、删除用户主题、打开 GitHub 和显示文件位置。Renderer 无 Node 权限，所有 IPC 参数都经过主进程白名单校验。

核心服务位于 `manager/core/`。主题仓库服务读取内置 `themes/catalog.json` 和用户数据目录中的声明式主题；包服务使用 `.wbskin` ZIP 格式，只允许一个 `theme.json` 与一个本地 PNG/JPG/WebP，拒绝 JavaScript、SVG、HTML、远程 URL、绝对路径、路径穿越、符号链接、重复文件、超量文件和超过上限的压缩包。运行服务调用现有 `scripts/injector.mjs`，使用同一个 `workbuddy-dream-skin-selection-v1` 状态键，确保管理器与 WorkBuddy 内部主题栏双向一致。

## 界面

视觉采用“桌面主题标本馆”方向：暖灰纸面为底、深墨标题、朱红状态点和高饱和主题缩略图。顶部是品牌与 WorkBuddy 状态，右侧提供“导入、设计主题、刷新、完全还原”；中部是响应式主题卡片网格，每张卡显示预览、名称、风格标签、当前状态和主要操作；底部固定显示安全说明及自动重启开关。界面避免常见紫色渐变模板，使用有节奏的非对称卡片、细线编号、纸张纹理和克制的进入动画。

## 数据流与错误处理

启动时主进程检测 WorkBuddy 路径、进程和 `127.0.0.1:9336` 调试端口，聚合内置主题、用户主题和当前选择后返回 Renderer。应用主题时先校验主题，再确保 WorkBuddy 调试端点可用；必要时经用户设置允许重启，随后执行一次注入并启动尊重当前选择的后台守护。恢复操作先停止精确匹配的注入器进程，再调用清理表达式，不修改 WorkBuddy 安装文件。

所有失败返回稳定错误码与中文提示，包括未安装、需要重启、调试端口未就绪、主题损坏、包不安全、文件过大、ID 冲突和注入校验失败。文件导入先解压到随机临时目录，通过所有校验后再原子移动到用户主题目录；中途失败必须清理临时文件并保留当前主题。

## 测试与交付

采用 Node 内置测试框架执行 TDD。自动测试覆盖目录聚合、主题包路径安全、资源限制、导入导出往返、自定义主题生成、状态持久化、IPC 白名单、Electron 安全设置、应用/恢复命令和 15 套主题目录。UI 测试检查可访问名称、键盘焦点、忙碌状态、响应式网格和错误提示。交付配置使用 electron-builder，Windows 生成 NSIS 安装版与 portable EXE，macOS 生成 DMG；Release 工作流继续提供对应产物。最终还需在真实 WorkBuddy 5.2.6 完成应用、切换、恢复及页面视觉验证。
