# WorkBuddy 内置主题切换栏设计

## 交互与视觉

切换器固定在 WorkBuddy 内容区右上角，采用参考图中的细长玻璃胶囊形态。桌面宽度足够时直接显示 8 个主题短名称以及“恢复原主题”；当前项使用该主题的主色与次色形成高亮，未选项保持半透明浅色或黑金主题下的深色玻璃。点击后立即更新 Hero、左侧菜单、按钮、输入区和页面底色，不重启 WorkBuddy。窄窗口允许横向滚动，并保留键盘焦点、`aria-pressed` 与清晰的 focus ring。切换器位于装饰层内，但自身恢复 `pointer-events: auto`，其他星光、人物和光球继续点击穿透。

## 数据流与状态

Injector 读取 `themes/catalog.json`，校验每套 `theme.json` 和图片路径后，将小型主题元数据目录与安全图片 URL 注入 Renderer。Renderer 创建一次切换栏，事件采用委托绑定，选择后更新根节点 CSS 变量、主题 ID、Hero 图片与按钮状态，并把选择保存到 WorkBuddy 的 localStorage。守护进程重复注入时优先读取已保存选择，避免把用户刚点击的主题改回启动默认值；桌面选择器首次启动仍可明确覆盖保存值。“恢复原主题”移除 `workbuddy-dream-skin` 根类和视觉元素，但保留切换栏，便于再次选择。桌面 Restore 会完整删除切换栏、样式和持久化状态。

## 安全与验证

目录仍经过现有 schema 校验，图片地址只能来自已验证的主题目录，用户点击只接受目录中存在的 ID，不执行主题 JavaScript。测试先覆盖目录载荷、按钮生成、选择持久化、原生主题状态和清理；随后在真实 WorkBuddy 中点击至少三套明暗主题并检查计算样式、Hero 宽度和活动按钮。最后验证重启后保留选择、恢复原主题不残留组件换色、桌面 Restore 可彻底移除，以及 Windows/macOS 脚本和全部自动测试仍通过。
