# 隐藏场景选择器与 GitHub 入口设计

## 目标

隐藏欢迎页中“日常办公 / 代码开发 / 设计创意”三个场景选择按钮，并在右上角主题栏加入指向 `https://github.com/buzhangsaner/workbuddy-skin` 的 GitHub 图标链接。

## 方案比较

1. 推荐：CSS 隐藏原生 `.wb-scene-tabs`，主题栏注入安全 `<a>` 与内联 SVG。改动最小、可恢复、不依赖网络图标资源，也不会影响已有主题按钮事件。
2. 删除原生节点并注入远程 GitHub 图标。删除节点可能被 WorkBuddy 重建，远程资源还会受网络与 CSP 影响。
3. 把 GitHub 链接放进自定义编辑器。实现简单，但入口不够明显，也不符合“右上角”的使用习惯。

采用方案 1。

## 交互与视觉

- `.wb-scene-tabs` 在 Dream Skin 欢迎页内不占布局空间。
- GitHub 入口位于“恢复原主题”之后，保持 32px 方形玻璃按钮、圆形悬停高亮和清晰键盘焦点。
- 链接拥有中文 `aria-label`、`title`、`target="_blank"` 与 `rel="noopener noreferrer"`。
- 窄屏继续随主题栏横向滚动，不单独覆盖页面内容。

## 安全与验证

- URL 是源码常量，不接受主题或用户输入。
- SVG 通过 DOM API 创建，不用 `innerHTML`，不加载远程脚本或图片。
- 自动测试覆盖隐藏规则、链接地址、可访问属性、SVG 和清理生命周期；实机验证图标命中、场景容器隐藏与主题切换不回归。
