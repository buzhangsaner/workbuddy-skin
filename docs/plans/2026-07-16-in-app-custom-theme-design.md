# WorkBuddy 内置自定义主题设计

## 目标与体验

在右上角主题切换栏加入“自定义”入口。点击后打开右侧抽屉式玻璃编辑器，不离开 WorkBuddy，也不打开外部浏览器。用户可以修改主题名称、主标题、英文眉题、主色、辅助色、背景色、文字色，并选择本地 PNG、JPG 或 WebP 背景图。每次输入都立即调用现有主题应用流程，因此侧栏、欢迎页、对话页、卡片和输入框同步预览。

编辑过程分为草稿和保存态。打开面板时记录当前主题；实时预览不写永久选择。点击“取消”或按 Escape 会恢复打开前主题。点击“保存主题”后，经过清洗的主题对象写入 `workbuddy-dream-custom-theme-v1`，当前选择写为 `custom`。下次注入或重启 WorkBuddy 时，Renderer 先读取并验证自定义主题，再恢复选择。右上角“自定义”按钮显示保存后的主题色。

## 安全与数据流

Renderer 继续运行在 WorkBuddy 的隔离沙箱内，不新增 Node、IPC、网络请求或安装包修改。文本移除控制字符并限制长度；颜色只允许六位十六进制；图片只接受 PNG/JPEG/WebP MIME，不接受 SVG。图片在浏览器 Canvas 中缩放到不超过 1920×1200，输出压缩 Data URL，并限制最终长度，避免填满 LocalStorage。

自定义主题使用固定 ID `custom`，不能覆盖内置 8 套主题。解析 LocalStorage 失败、字段非法或图片不合规时回退到安全默认自定义主题，不影响内置主题。面板使用原生 label、input、button 和 dialog 语义，支持键盘焦点、Escape、取消、保存和恢复默认。

## 视觉方向

编辑器采用“主题调色台”而不是普通设置表单：右侧 360px 半透明抽屉、低饱和纸张纹理、紧凑色卡、醒目的图片投放区和固定底部操作栏。它与当前沉浸式主题共享颜色变量，但通过独立高层交互根节点保证可点击，且不会被装饰层的 `pointer-events:none` 影响。窄屏下改为底部全宽面板，主题切换栏继续横向滚动。
