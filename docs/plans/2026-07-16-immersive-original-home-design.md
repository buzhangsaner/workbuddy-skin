# WorkBuddy 原版沉浸式首页设计

## 视觉构图

欢迎页从“顶部横幅＋中间原生表单”升级为整页主题舞台。主题图片从侧栏右边缘铺到工作区底部，保留人物或角色在右上至右中的视觉重心；左侧约三分之一放主题品牌眉题、主标题、说明和状态胶囊。原生场景按钮位于标题区下方，四个原生快捷操作改为横向玻璃卡片阵列，输入区固定在主内容下段，形成与参考原版相同的阅读路径。右上主题切换栏保持独立 `no-drag` 交互层。每套主题仍使用自己的主色、次色、表面色和字体色；黑金使用深色玻璃，其余主题使用半透明浅色玻璃。

## 原生功能与数据流

Renderer 在现有装饰层中新建 `wb-dream-immersive-copy`，内容只来自已校验主题的 `name`、`tagline` 与 `quote`，不接受 HTML。WorkBuddy 的场景按钮、快捷操作和 Composer 不复制、不伪造，CSS 只重新定位稳定 class。主题切换时 `applyTheme` 同步更新全幅图片、品牌标题与文案；选择原生主题时隐藏整个沉浸式舞台并恢复 WorkBuddy 原布局。Hash 路由离开 Chat 后，Hero 与品牌区自动隐藏，其他工作区继续使用轻量色板。

## 响应式与验证

宽屏采用左文右图的完整构图；中等宽度压缩标题字号、卡片间距和输入区宽度；小于 980px 时标题区降低透明度并收紧，低于 760px 时隐藏人物背景和品牌大标题，回退为当前紧凑布局。自动测试覆盖舞台 DOM、纯文本更新、全视口 Hero、稳定组件映射、原生主题回退与路由隔离。实机验证使用 1920px 和 1200px 两种窗口，重点检查粉系、灵感、紫夜、黑金四个差异最大的主题，再遍历其余主题确认主体不遮挡主题栏和输入区。
